# Contributing to Aura

First off, thank you for considering contributing to Aura! It's people like you that make Aura such a great tool.

## How Can I Contribute?

### Reporting Bugs

This section guides you through submitting a bug report for Aura. Following these guidelines helps maintainers and the community understand your report, reproduce the behavior, and find related reports.

* **Check the documentation** to see if you are using the language correctly.
* **Search the Issue Tracker** to see if the bug has already been reported.
* If you find a bug, **open a new issue** using the Bug Report template.

### Suggesting Enhancements

This section guides you through submitting an enhancement suggestion for Aura, including completely new features and minor improvements to existing functionality.

* **Check if there is already a similar suggestion**.
* **Open a new issue** using the Feature Request template.

### Your First Code Contribution

1. Fork the repository.
2. Clone your fork: `git clone https://github.com/YOUR_USERNAME/Aura-Programming-Language.git`
3. Create a branch: `git checkout -b feature/your-feature-name`
4. Install dependencies: `pip install -e .`
5. Make your changes.
6. Run tests: `pytest` (if available)
7. Commit your changes: `git commit -m "Add some feature"`
8. Push to your fork: `git push origin feature/your-feature-name`
9. Open a Pull Request.

## Styleguides

### Python Coding Standards

* We use `black` for formatting.
* Follow [PEP 8](https://www.python.org/dev/peps/pep-0008/).

### Documentation

* Documentation is located in the `docs/` directory.
* Use Markdown for all documentation files.

## Community

* Join our [Discord/Slack/Discussions] (if available).
* Follow our [Code of Conduct](CODE_OF_CONDUCT.md).
